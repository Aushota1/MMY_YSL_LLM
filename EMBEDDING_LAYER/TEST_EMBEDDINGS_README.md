# 🔍 Тестирование Embeddings - Поиск похожих слов

Скрипт `test_embeddings.py` позволяет проверить работу обученных embeddings, найдя ближайшие по контексту слова для заданного слова или предложения.

## 📋 Описание

Скрипт использует обученную модель embeddings (`.pth` файл) и токенизатор (`.pkl` файл) для:
- Получения embedding вектора для входного слова или предложения
- Вычисления косинусного сходства со всеми токенами в словаре
- Вывода топ-N ближайших слов по семантическому сходству

## 🚀 Использование

### Интерактивный режим (рекомендуется)

```bash
python test_embeddings.py
```

Скрипт запросит:
1. Путь к файлу модели (по умолчанию: `embedding_model.pth`)
2. Путь к файлу токенизатора (по умолчанию: `chekpoint.pkl`)
3. Затем можно вводить слова/предложения для поиска похожих слов

**Пример сессии:**
```
🔍 Введите запрос: привет
Количество результатов [по умолчанию: 10]: 15
Использовать позиционное кодирование? [y/N]: n

================================================================================
🔍 Поиск похожих слов для: 'привет'
================================================================================

📊 Топ-15 ближайших слов:

Ранг   Слово                           Косинусное сходство
------------------------------------------------------------
1      здравствуй</w>                           0.8234
2      добрый</w>                                0.7891
3      день</w>                                  0.7654
...
```

### Режим командной строки

```bash
# Базовое использование
python test_embeddings.py --text "привет" --top-k 10

# С указанием путей к файлам
python test_embeddings.py --model embedding_model.pth --tokenizer chekpoint.pkl --text "машина" --top-k 20

# С использованием позиционного кодирования
python test_embeddings.py --text "привет мир" --positional --top-k 15
```

**Параметры командной строки:**
- `--model` - Путь к файлу модели `.pth` (по умолчанию: `embedding_model.pth`)
- `--tokenizer` - Путь к файлу токенизатора `.pkl` (по умолчанию: `chekpoint.pkl`)
- `--text` - Текст для поиска (если не указан, запускается интерактивный режим)
- `--top-k` - Количество ближайших слов (по умолчанию: 10)
- `--positional` - Использовать позиционное кодирование (по умолчанию: выключено)

## 📊 Как это работает

1. **Загрузка модели**: Скрипт загружает обученную модель и токенизатор
2. **Кодирование текста**: Входной текст кодируется через BPE токенизатор
3. **Получение embedding**: 
   - Для одиночных слов: используется embedding первого токена
   - Для предложений: используется среднее значение embeddings всех токенов
   - По умолчанию используется только token embedding (без позиционного кодирования) для лучшего семантического поиска
4. **Поиск похожих слов**: Вычисляется косинусное сходство со всеми токенами в словаре
5. **Вывод результатов**: Показываются топ-N ближайших слов с их косинусным сходством

## ⚙️ Параметры поиска

### Позиционное кодирование

- **Без позиционного кодирования** (по умолчанию): Лучше для поиска семантически похожих слов
  - Используется только token embedding
  - Игнорируется позиция слова в предложении
  - Рекомендуется для поиска синонимов и связанных слов

- **С позиционным кодированием**: Учитывает позицию слова
  - Используется полный embedding layer с позиционной информацией
  - Может быть полезно для анализа контекста в предложениях
  - Обычно менее эффективно для поиска похожих слов

### Исключение специальных токенов

По умолчанию специальные токены (`<PAD>`, `<UNK>`, `<BOS>`, `<EOS>`, `<SEP>`) исключаются из результатов поиска.

## 💡 Примеры использования

### Поиск синонимов

```bash
python test_embeddings.py --text "хороший" --top-k 10
```

### Поиск связанных слов
```bash
python test_embeddings.py --text "машина" --top-k 15
```

### Анализ предложений
```bash
python test_embeddings.py --text "привет как дела" --positional --top-k 20
```

## 🔧 Требования

- PyTorch
- Обученная модель embeddings (`.pth` файл)
- Обученный токенизатор (`.pkl` файл)
- Модули проекта: `BPE_STUCTUR`, `EMBEDDING_LAYER`

## 📝 Примечания

- Косинусное сходство варьируется от -1 до 1, где 1 означает идентичные векторы
- Чем выше значение, тем ближе слова по семантике
- Результаты зависят от качества обучения модели
- Для лучших результатов рекомендуется обучение на большом корпусе текстов

## ❓ Решение проблем

**Ошибка: "Файл модели не найден"**
- Убедитесь, что файл `.pth` существует в указанном пути
- Проверьте, что модель была сохранена через `embedding_trainer.py`

**Ошибка: "Не удалось закодировать текст"**
- Убедитесь, что токенизатор обучен на похожих данных
- Проверьте, что текст содержит слова из словаря токенизатора

**Низкое качество результатов**
- Модель может быть недостаточно обучена
- Попробуйте увеличить количество эпох обучения
- Убедитесь, что размер embedding достаточен (256+)
