# Создание датасета для обучения Embeddings

## 📋 Описание

Скрипт `create_embedding_dataset.py` создает большой датасет для обучения embeddings, объединяя существующие текстовые файлы, очищая текст и подготавливая данные в правильном формате.

## 🎯 Возможности

✅ **Загрузка из файлов** - Объединение нескольких текстовых файлов  
✅ **Загрузка из директорий** - Рекурсивная загрузка всех текстовых файлов  
✅ **Очистка текста** - Нормализация и удаление лишних символов  
✅ **Разбиение на предложения** - Умное разбиение текста на предложения  
✅ **Фильтрация** - Удаление слишком коротких/длинных предложений  
✅ **Удаление дубликатов** - Очистка от повторяющихся предложений  
✅ **Синтетическая генерация** - Создание дополнительных текстов  
✅ **Статистика** - Подробная статистика созданного датасета  

## 🚀 Быстрый старт

### Базовое использование

```bash
# Использование файла war_and_peace.ru.txt (если существует)
python create_embedding_dataset.py

# Указание конкретных файлов
python create_embedding_dataset.py --files war_and_peace.ru.txt text1.txt text2.txt

# Загрузка из директории
python create_embedding_dataset.py --dirs ./texts ./documents

# Комбинирование файлов и директорий
python create_embedding_dataset.py --files file1.txt --dirs ./texts
```

### Расширенные опции

```bash
# С синтетической генерацией (увеличение датасета в 2 раза)
python create_embedding_dataset.py --files war_and_peace.ru.txt --synthetic

# С добавлением примеров текстов
python create_embedding_dataset.py --files war_and_peace.ru.txt --add-samples

# Настройка фильтрации по длине
python create_embedding_dataset.py \
    --files war_and_peace.ru.txt \
    --min-length 15 \
    --max-length 300

# Полный пример с всеми опциями
python create_embedding_dataset.py \
    --files war_and_peace.ru.txt text1.txt \
    --dirs ./texts \
    --output my_dataset.txt \
    --min-length 10 \
    --max-length 500 \
    --synthetic \
    --synthetic-multiplier 3 \
    --add-samples
```

## 📊 Параметры

### Основные параметры

- `--files` - Список файлов для загрузки (можно указать несколько)
- `--dirs` - Список директорий для загрузки (рекурсивно)
- `--output` - Выходной файл (по умолчанию: `embedding_dataset.txt`)

### Фильтрация

- `--min-length` - Минимальная длина предложения в словах (по умолчанию: 10)
- `--max-length` - Максимальная длина предложения в словах (по умолчанию: 500)

### Генерация

- `--synthetic` - Использовать синтетическую генерацию для увеличения датасета
- `--synthetic-multiplier` - Множитель для синтетических текстов (по умолчанию: 2)
- `--add-samples` - Добавить примеры текстов разных типов

### Дополнительные опции

- `--no-shuffle` - Не перемешивать предложения
- `--no-deduplicate` - Не удалять дубликаты

## 📝 Формат выходного файла

Выходной файл содержит одно предложение на строку:

```
Машинное обучение представляет собой подраздел искусственного интеллекта.
Нейронные сети используются для решения сложных задач распознавания образов.
Глубокое обучение требует больших вычислительных ресурсов и данных.
...
```

## 🔧 Обработка текста

### Очистка

Скрипт автоматически:
- Удаляет множественные пробелы
- Нормализует кавычки
- Удаляет специальные символы (кроме пунктуации)
- Исправляет пробелы вокруг пунктуации

### Разбиение на предложения

Умное разбиение, учитывающее:
- Сокращения (т.д., т.п., и т.д., и т.п., и др.)
- Аббревиатуры (г., гг., в., вв., руб., коп.)
- Временные обозначения (мин., сек., ч., сут.)
- Английские сокращения (Mr., Mrs., Dr., etc.)

## 💡 Рекомендации

### Размер датасета

- **Минимальный**: 1,000-5,000 предложений (для тестирования)
- **Рекомендуемый**: 10,000-50,000 предложений (для хорошего качества)
- **Оптимальный**: 50,000+ предложений (для production)

### Длина предложений

- **Короткие** (10-30 слов): Хорошо для быстрого обучения
- **Средние** (30-100 слов): Оптимально для большинства задач
- **Длинные** (100+ слов): Для сложных контекстных задач

### Синтетическая генерация

Используйте `--synthetic` для:
- Увеличения размера датасета
- Создания вариаций текстов
- Улучшения обобщающей способности

⚠️ **Внимание**: Синтетическая генерация может снизить качество, если переусердствовать.

## 🐛 Решение проблем

### Проблема: Файл не найден

```
⚠️  Файл не найден: file.txt
```

**Решение**: Проверьте путь к файлу, используйте абсолютный путь или убедитесь, что файл существует.

### Проблема: Кодировка

Если файл содержит нечитаемые символы, скрипт автоматически игнорирует ошибки кодировки. Для явного указания кодировки измените код.

### Проблема: Слишком мало предложений

Если после фильтрации осталось мало предложений:
- Уменьшите `--min-length`
- Увеличьте `--max-length`
- Используйте `--synthetic` для увеличения
