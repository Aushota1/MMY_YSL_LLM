# Система анализа ранга и предсказания цены на основе классификатора с эмбеддингами и TA-Lib

Подробное описание использования архитектуры «эмбеддинги → фиксированный вектор → классификатор» для торговли фьючерсами и спотом с учётом комиссий и быстрых сделок. Интеграция с библиотекой **TA-Lib** для технического анализа.

---

## 1. Идея переноса вашей системы на торговлю

### 1.1 Текущая система (tokenier)

- **Вход:** текст RTL-модуля (последовательность токенов).
- **Представление:** токенизатор → слой эмбеддингов → массив (n, d) → агрегация (mean/max/sequence) → вектор фиксированной размерности (например 256).
- **Выход:** классификатор (SVM, XGBoost, RF и т.д.) по вектору → метка класса (тип модуля).

### 1.2 Аналогия для рынка

| Компонент tokenier | Торговая интерпретация |
|--------------------|------------------------|
| «Текст» / последовательность | Временное окно рыночного состояния: OHLCV + индикаторы TA-Lib по каждому бару (или тику). |
| «Токены» | Отдельные бары (свечи) или сжатые представления бара (признаки за бар). |
| «Эмбеддинги» | Векторное представление одного бара или окна: либо сырые признаки (OHLCV + Talib), либо обученный слой (embedding) по «токенам» окна. |
| «Фиксированный вектор» | Один вектор на окно: агрегация по времени (mean/max/sequence) или конкатенация скользящих признаков. |
| «Класс» | Направление/ранг: например «вверх», «вниз», «флэт» или «сильный рост», «рост», «нейтрал», «падение», «сильное падение» (ранг от 1 до 5). |

Идея: **рыночное окно → представление (признаки Talib + опционально обученные эмбеддинги по последовательности) → фиксированный вектор → классификатор → сигнал/ранг для решения о сделке**.

---

## 2. Роль TA-Lib в формировании признаков

### 2.1 Назначение TA-Lib

- **TA-Lib** (Technical Analysis Library) даёт 150+ индикаторов по OHLCV.
- Используется как источник **признаков** для векторизации состояния рынка в каждый момент (бар) и по окну.

### 2.2 Основные группы индикаторов (для входа в классификатор)

| Группа | Примеры функций | Назначение |
|--------|------------------|------------|
| Overlap Studies | `SMA`, `EMA`, `BBANDS`, `WMA` | Тренд, волатильность (полосы). |
| Momentum | `RSI`, `MACD`, `STOCH`, `CCI`, `MOM`, `WILLR` | Перекупленность/перепроданность, импульс. |
| Volume | `OBV`, `AD`, `ADOSC` | Подтверждение движения объёмом. |
| Volatility | `ATR`, `NATR`, `TRANGE` | Размер движения, стопы. |
| Pattern Recognition | `CDLDOJI`, `CDLENGULFING`, … | Свечные паттерны (опционально). |

Типичные вызовы (Python):

```python
import talib
import numpy as np

# real = close (или open, high, low)
rsi = talib.RSI(close, timeperiod=14)
macd, macdsignal, macdhist = talib.MACD(close, 12, 26, 9)
upper, middle, lower = talib.BBANDS(close, 5, 2, 2)
atr = talib.ATR(high, low, close, timeperiod=14)
obv = talib.OBV(close, volume)
```

Все индикаторы в начале ряда дают NaN — нужна обрезка или заполнение при построении датасета.

### 2.3 Формирование «токена» одного бара

Для одного бара (одной свечи) можно собрать вектор признаков:

- Нормализованные OHLCV (например, log-return close, range high-low, объём в долях от скользящей средней объёма).
- Выходы TA-Lib за этот бар: RSI, MACD histogram, положение цены относительно BBANDS, ATR (нормализованный) и т.д.

Этот вектор можно трактовать как **«токен»** одного момента времени. Окно из N баров даёт последовательность из N таких «токенов».

## 3. Два варианта архитектуры

### 3.1 Вариант A: Без обученных эмбеддингов (только признаки TA-Lib)

- На каждом баре считаются индикаторы TA-Lib + базовые производные (returns, range).
- Окно из последних N баров → либо **конкатенация** всех признаков по окну (один большой вектор), либо **агрегация** по времени (mean/max/std по каждому признаку за окно) → фиксированный вектор.
- Этот вектор подаётся в **тот же пайплайн**, что и в Learn_ML: StandardScaler (при необходимости) → классификатор (SVM, XGBoost, RF и т.д.).
- Плюсы: простота, интерпретируемость, не нужна нейросеть. Минусы: нет обучения представления временной структуры.

### 3.2 Вариант B: С «эмбеддингами» по последовательности (идея вашей системы)

- Каждый бар кодируется вектором признаков (OHLCV + Talib) фиксированной длины — «токен».
- Окно из N баров → матрица (N, D). Далее:
  - **Вариант B1:** обучить слой эмбеддингов (или маленькую 1D-CNN / LSTM), который переводит (N, D) в (N, d), затем применить **агрегацию** (mean/max) как в `embedding_to_fixed_vector` → вектор длины d (или 256) → классификатор.
  - **Вариант B2:** без обучения: считать (N, D) уже «последовательностью эмбеддингов» и применять к ней mean/max по оси 0 → вектор D (или после отбора признаков меньше) → классификатор.

В обоих случаях сохраняется идея: **последовательность → одно фиксированное представление → классификатор**.

## 4. Анализ ранга и предсказание цены

### 4.1 Ранг

- **Ранг** — упорядоченная оценка ожидаемой доходности или силы сигнала.
- Пример: классы `rank_1` (сильное ожидание роста) … `rank_5` (сильное ожидание падения). Метка на момент t может задаваться по будущей доходности за горизонт H (например, return за следующие 5 минут или 1 бар).
- Разбиение на бакеты: по перцентилям будущего return (например, 5 квантилей) или по порогам (сильный рост / рост / флэт / падение / сильное падение).
- Классификатор обучается предсказывать этот ранг; на инференсе выдаётся класс (ранг), по которому можно:
  - ранжировать инструменты (выбрать лучший для лонга/шорта),
  - ранжировать моменты времени (когда входить),
  - фильтровать сделки (входить только при rank_1 или rank_5).

### 4.2 Предсказание цены (как классификация)

- Прямое предсказание цены (регрессия) можно заменить **классификацией по бакетам** будущего изменения цены или доходности.
- Пример: целевая переменная — «направление через 1 бар»: up / flat / down (или 5 классов по размеру движения). Тогда «предсказание цены» интерпретируется как выбор класса; средняя цена по классу может использоваться как точечная оценка.

### 4.3 Спот и фьючерсы

- **Спот:** комиссия обычно в процентах от объёма; проскальзывание при быстрых сделках существенно.
- **Фьючерсы:** комиссия за сделку (фикс или за контракт), маржа, возможно более узкий спред и большая ликвидность на популярных контрактах — удобно для быстрых входов/выходов.
- Один и тот же пайплайн (признаки Talib → вектор → классификатор → ранг/класс) применим к обоим; различаются только:
  - способ учёта комиссии и проскальзывания в бэктесте,
  - размер позиции и правила риска (маржа у фьючерсов).

## 5. Учёт комиссии и быстрых сделок

### 5.1 Комиссия

- **Спот:** часто комиссия в % от оборота (например 0.1% на вход и 0.1% на выход → 0.2% за круг).
- **Фьючерсы:** за контракт или за сделку (вход + выход).
- В бэктесте на каждый вход и выход вычитается комиссия из PnL. Для быстрых сделок число сделок велико → суммарная комиссия может съесть преимущество стратегии.

Формула для спота (упрощённо):

- `cost_per_round_trip = 2 * commission_pct * notional`
- Или с разными комиссиями на вход/выход: `cost_in + cost_out`.

Для фьючерсов:

- `cost_per_round_trip = commission_per_side * 2 * num_contracts` (или по объёму, в зависимости от тарифа).

### 5.2 Быстрые сделки (скальпинг, короткий горизонт)

- Короткий период удержания (несколько баров или тиков) → много сделок в день → комиссия и проскальзывание критичны.
- Нужно:
  - в бэктесте использовать **реалистичные** комиссии и модель проскальзывания (например, фиксированный сдвиг цены входа/выхода или доля от ATR);
  - фильтровать сигналы: входить только когда предсказанный ранг (или ожидаемый return) достаточно высок, чтобы перекрыть комиссию и типичное проскальзывание.
- Минимальный целевой доход за сделку (в долях):  
  `min_target_return >= 2 * commission_pct + slippage_estimate`.

### 5.3 Рекомендации в коде

- Параметры: `commission_pct` (спот) или `commission_per_trade` (фьючерсы), `slippage_bps` (в базисных пунктах) или `slippage_atr_fraction`.
- На каждый выход из позиции в симуляции:
  - вычесть комиссию из PnL;
  - при необходимости сдвинуть цену выхода на величину проскальзывания (ухудшить исполнение).
- Считать метрики после комиссий: net PnL, Sharpe, max drawdown, число сделок.

## 6. Пайплайн данных и обучения (конкретно)

### 6.1 Исходные данные

- **OHLCV** по таймфрейму (например 1m, 5m для быстрых сделок) для спота или фьючерса.
- При необходимости — несколько инструментов для ранжирования.

### 6.2 Шаг 1: Расчёт признаков TA-Lib

- По каждому бару: RSI, MACD (и histogram), BBANDS (нормализованное положение цены между полосами), ATR, OBV (или его производная), при необходимости свечные паттерны.
- Обработка NaN: обрезать начальные строки или заполнить (например, forward fill только после первого валидного значения).
- Дополнительно: log-return close, range (high-low), объём относительно скользящей средней объёма.

### 6.3 Шаг 2: Целевая переменная (ранг / класс)

- Горизонт H (например, 1 бар вперёд или 5 баров).
- `forward_return = (close[t+H] - close[t]) / close[t]` (или log-return).
- Разбиение на классы:
  - по перцентилям (например, 5 классов),
  - или по порогам: сильный рост / рост / флэт / падение / сильное падение.
- Метка привязывается к бару t (по состоянию на t предсказываем класс, соответствующий return за [t, t+H]).

### 6.4 Шаг 3: Окно и вектор

- Для каждого t формируется окно [t-L, t] (L баров).
- **Вариант A:** конкатенация или агрегация (mean/max) признаков по окну → один вектор длины `L * D` или `D` (при агрегации по времени).
- **Вариант B:** матрица (L, D) → (опционально) слой эмбеддингов/CNN/LSTM → агрегация → вектор (например 256) → классификатор.

### 6.5 Шаг 4: Обучение классификатора

- Датасет: строки = моменты времени t, столбцы = признаки (f0, f1, …) + `target` (ранг/класс).
- Разбиение: по времени (не shuffle!), например train = первые 70%, test = последние 30%, чтобы не было утечки из будущего.
- Нормализация: StandardScaler по train, применить к test.
- Обучение: тот же код, что в `Learn_ML/train_classifiers.py` (SVM, XGBoost, RF и т.д.), сохранение model + scaler + label_encoder в joblib.

### 6.6 Шаг 5: Инференс и торговый сигнал

- В момент t: текущее окно → вектор → scaler → модель → предсказанный класс (ранг).
- Правило: например, входить в лонг при rank_1, в шорт при rank_5; или ранжировать инструменты и торговать топ-1.
- Выход: по таймеру (N баров), по стоп-лоссу/тейк-профиту или по смене ранга.

## 8. Пример структуры кода с TA-Lib

```python
import numpy as np
import pandas as pd
import talib
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import TimeSeriesSplit
import joblib

def compute_talib_features(df: pd.DataFrame) -> pd.DataFrame:
    """OHLCV -> TA-Lib индикаторы."""
    o, h, l, c, v = df['open'].values, df['high'].values, df['low'].values, df['close'].values, df['volume'].values
    out = pd.DataFrame(index=df.index)
    out['rsi'] = talib.RSI(c, 14)
    out['macd'], out['macd_sig'], out['macd_hist'] = talib.MACD(c, 12, 26, 9)
    out['bb_upper'], out['bb_mid'], out['bb_lower'] = talib.BBANDS(c, 20, 2, 2)
    out['atr'] = talib.ATR(h, l, c, 14)
    out['obv'] = talib.OBV(c, v)
    out['ret'] = np.log(c / np.roll(c, 1))
    out.loc[0, 'ret'] = np.nan
    out['range_hl'] = (h - l) / (c + 1e-8)
    return out

def build_windows(features: pd.DataFrame, target_col: str, window_len: int, horizon: int, n_quantiles: int = 5):
    """Окна и целевой ранг по квантилям forward return."""
    features = features.dropna(how='all').ffill().dropna()
    # Доходность за horizon баров вперёд: сумма log-return с t+1 по t+horizon
    fwd = sum(features['ret'].shift(-i) for i in range(1, horizon + 1))
    fwd = fwd.dropna()
    labels = pd.qcut(fwd.rank(method='first'), n_quantiles, labels=False, duplicates='drop')
    X, y = [], []
    for i in range(window_len, len(features) - horizon):
        if i not in labels.index:
            continue
        X.append(features.iloc[i - window_len:i].values.flatten())
        y.append(labels.loc[i])
    return np.array(X), np.array(y)

def train_and_save(X_train, y_train, X_test, y_test, path_model='trading_classifier.joblib'):
    scaler = StandardScaler()
    X_train_s = scaler.fit_transform(X_train)
    X_test_s = scaler.transform(X_test)
    from sklearn.ensemble import RandomForestClassifier
    clf = RandomForestClassifier(n_estimators=100)
    clf.fit(X_train_s, y_train)
    bundle = {'model': clf, 'scaler': scaler}
    joblib.dump(bundle, path_model)
    return clf.predict(X_test_s)
```

- В бэктесте для каждого бара вызывать предсказание ранга, применять правило входа/выхода, считать PnL с вычетом комиссии и проскальзывания.

## 9. Бэктест и метрики

- **Временное разбиение:** train до даты T, test после T (или walk-forward).
- **Симуляция:** на каждом баре теста — предсказание ранга; при срабатывании правила — открытие/закрытие позиции; при закрытии: PnL = (price_exit - price_entry) * side - commission - slippage.
- **Метрики:** суммарный PnL, Sharpe ratio (по дневным или сделкам), max drawdown, количество сделок, средний PnL на сделку после комиссий.

---

## 10. Риски и ограничения

- **Переобучение:** много признаков TA-Lib при малом объёме данных; нужна регуляризация, отбор признаков или упрощение модели.
- **Режим рынка:** классификатор, обученный на одном режиме, может деградировать в другом; полезны периодическое переобучение или адаптация.
- **Задержки и ликвидность:** быстрые сделки требуют низкой задержки исполнения; на споте проскальзывание может быть большим при низкой ликвидности.
- **Комиссии:** при высокой частоте сделок стратегия может быть убыточной после комиссий; обязательно тестировать с реалистичными комиссиями и проскальзыванием.

## 11. Краткая схема системы

```
[OHLCV] → TA-Lib (RSI, MACD, BBANDS, ATR, OBV, …) → признаки по барам
       → окно из L баров → вектор (конкатенация или агрегация; опционально эмбеддинг по последовательности)
       → StandardScaler → классификатор (SVM/XGB/RF) → ранг/класс
       → торговое правило (вход/выход) → симуляция с комиссией и проскальзыванием → PnL и метрики.
```
