# Консольное приложение для обучения BPE токенизатора

## Описание

`tokenizer_trainer.py` - удобное консольное приложение для работы с BPE токенизатором. Позволяет обучать, дообучать и тестировать токенизатор через интуитивный интерфейс.

## Основные возможности

### ✨ Новые функции

1. **Дообучение токенизатора** - автоматическое расширение словаря при обучении на новых данных
2. **Чекпоинты и прерывание** ⭐ НОВОЕ - безопасное прерывание обучения с сохранением прогресса
3. **Удобный консольный интерфейс** - интерактивное меню для всех операций
4. **Работа с файлами** - обучение и дообучение на текстовых файлах
5. **Интерактивное тестирование** - быстрая проверка работы токенизатора

## Запуск

```bash
python tokenizer_trainer.py
```

## Меню приложения

### 1. Создать новый токенизатор
Создает новый экземпляр токенизатора с указанным размером словаря.

### 2. Загрузить существующую модель
Загружает ранее сохраненную модель из файла `.pkl`.

### 3. Обучить токенизатор (с нуля)
Обучение на введенных вручную текстах или корпусе.

### 4. Дообучить токенизатор (расширить словарь) ⭐ НОВОЕ
**Автоматическое расширение словаря** при обучении на новых данных:
- Сохраняет существующие слияния
- Добавляет новые токены из нового корпуса
- Продолжает обучение с существующими правилами
- Можно ограничить количество новых слияний

### 5. Обучить на файле
Обучение на текстовом файле (каждая строка = один текст).

### 6. Дообучить на файле ⭐ НОВОЕ
Дообучение на новом текстовом файле с автоматическим расширением словаря.

### 7. Тестировать токенизатор
Быстрое тестирование на одном тексте с выводом токенов.

### 8. Показать информацию о модели
Показывает:
- Размер словаря
- Количество слияний
- Специальные токены
- Примеры токенов

### 9. Сохранить модель
Сохранение обученной модели в файл `.pkl`.

### 10. Интерактивное тестирование
Режим для последовательного тестирования нескольких текстов.

## Примеры использования

### Пример 1: Обучение с нуля

1. Запустите приложение: `python tokenizer_trainer.py`
2. Выберите `1` - Создать новый токенизатор
3. Введите размер словаря (например, 5000)
4. Выберите `3` - Обучить токенизатор
5. Введите тексты для обучения (каждый с новой строки)
6. Введите пустую строку для завершения
7. Выберите `9` - Сохранить модель

### Пример 2: Дообучение существующей модели

1. Запустите приложение
2. Выберите `2` - Загрузить существующую модель
3. Введите путь к файлу модели
4. Выберите `4` - Дообучить токенизатор
5. Введите новые тексты для дообучения
6. (Опционально) Укажите максимальное количество новых слияний
7. Сохраните обновленную модель

### Пример 3: Обучение на файле

1. Подготовьте текстовый файл (например, `corpus.txt`)
2. Запустите приложение
3. Создайте токенизатор (`1`)
4. Выберите `5` - Обучить на файле
5. Введите путь к файлу
6. Сохраните модель

### Пример 4: Дообучение на файле

1. Загрузите существующую модель (`2`)
2. Выберите `6` - Дообучить на файле
3. Введите путь к новому файлу с данными
4. Модель автоматически расширит словарь новыми токенами
5. Сохраните обновленную модель

## Особенности дообучения

### Как работает дообучение:

1. **Сохранение существующих знаний**:
   - Все существующие слияния сохраняются
   - Порядок применения слияний сохраняется
   - Существующие токены остаются в словаре

2. **Добавление новых токенов**:
   - Новые символы из корпуса автоматически добавляются
   - Создаются новые слияния на основе новых данных
   - Словарь расширяется до указанного максимума

3. **Гибкость**:
   - Можно ограничить количество новых слияний
   - Можно дообучать несколько раз подряд
   - Работает с любым размером нового корпуса

### Пример сценария:

```
Шаг 1: Обучаем на русских текстах → словарь 5000 токенов
Шаг 2: Дообучаем на английских текстах → словарь 7000 токенов
Шаг 3: Дообучаем на технических терминах → словарь 8000 токенов
```

## Формат файлов

### Текстовый файл для обучения:
- Каждая строка = один текст
- Поддерживается UTF-8 кодировка
- Можно использовать любые разделители строк

Пример `corpus.txt`:
```
Мама мыла раму.
Привет, как дела?
Сегодня хорошая погода.
```

### Файл модели:
- Формат: `.pkl` (pickle)
- Содержит: словарь, слияния, порядок слияний, настройки

## Чекпоинты и прерывание обучения ⭐ НОВОЕ

### Что это?

Теперь вы можете безопасно прервать обучение (Ctrl+C) без потери прогресса! Токенизатор автоматически сохраняет текущее состояние.

### Как использовать:

1. **При обучении укажите путь для чекпоинтов:**
   ```
   Путь для сохранения чекпоинтов: checkpoint.pkl
   Сохранять чекпоинт каждые N итераций: 100
   ```

2. **Прервите обучение в любой момент:**
   - Нажмите `Ctrl+C`
   - Прогресс автоматически сохранится
   - Вы увидите информацию о сохраненном состоянии

3. **Продолжите обучение:**
   - Загрузите сохраненный чекпоинт
   - Продолжите обучение с того же места

### Пример использования:

```python
from BPE_STUCTUR import BPETokenizer

tokenizer = BPETokenizer(vocab_size=30000)

try:
    # Обучение с автоматическими чекпоинтами каждые 100 итераций
    tokenizer.train(
        corpus, 
        verbose=True,
        checkpoint_path="checkpoint.pkl",  # Автосохранение
        checkpoint_interval=100
    )
except KeyboardInterrupt:
    print("Обучение прервано, но прогресс сохранен!")
    # Модель уже сохранена в checkpoint.pkl
```

### Продолжение с чекпоинта:

```python
# Загружаем чекпоинт
tokenizer = BPETokenizer()
tokenizer.load("checkpoint.pkl")

# Продолжаем обучение
tokenizer.train(
    new_corpus,
    checkpoint_path="checkpoint.pkl",
    checkpoint_interval=100
)
```

## Советы по использованию

1. **Размер словаря**: 
   - Для небольших проектов: 1000-5000
   - Для средних: 10000-30000
   - Для больших: 50000+

2. **Дообучение**:
   - Используйте для добавления новых доменов
   - Не переобучайте на тех же данных
   - Сохраняйте модель после каждого дообучения

3. **Чекпоинты**:
   - **Всегда используйте чекпоинты** при длительном обучении
   - Рекомендуемый интервал: 100 итераций
   - При прерывании прогресс не потеряется
   - Можно продолжить обучение с любого чекпоинта

4. **Производительность**:
   - Большие файлы обрабатываются построчно
   - Используйте `verbose=False` для быстрого обучения
   - Сохраняйте промежуточные результаты через чекпоинты

## Решение проблем

### Ошибка: "Токенизатор должен быть обучен"
→ Сначала обучите токенизатор (`3` или `5`), затем используйте дообучение.

### Ошибка: "Файл не найден"
→ Проверьте путь к файлу. Используйте абсолютный путь или путь относительно текущей директории.

### Медленное обучение
→ Уменьшите размер словаря или используйте меньший корпус для тестирования.

## Технические детали

### Метод `continue_training()`

Новый метод в `BPETokenizer`:
- Принимает новый корпус текстов
- Сохраняет существующие слияния
- Добавляет новые символы в словарь
- Создает новые слияния на основе новых данных
- Обновляет порядок слияний

### Алгоритм дообучения:

1. Предобработка новых текстов
2. Извлечение новых символов
3. Применение существующих слияний к новым словам
4. Создание новых слияний
5. Обновление словаря
